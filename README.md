# Project 펌웨어 작동 원리 및 핀맵 문서

이 문서는 아두이노(ESP32) 기반 프로젝트의 하드웨어 핀 할당, 프로젝트 폴더 구조, 그리고 핵심 동작 방식을 설명합니다.

## 1. 폴더 구조 (`src` & `include`)
코드는 기능을 직관적으로 파악하고 새 모드를 편리하게 추가할 수 있도록, 시스템 별로 분할 조립되어 있습니다.

```text
End_p/
 ├── End_p.ino          : 아두이노 IDE 메인 스케치 파일 (setup과 loop의 큰 흐름만 제어)
 ├── include/           : 헤더 폴더 (함수의 뼈대와 환경 설정값)
 │    ├── pins.h        : 모든 하드웨어 핀 번호 스펙 시트
 │    ├── globals.h     : 현재 작동 상태(isRunning 등) 공유 및 튜닝 타이밍 제어
 │    ├── actuator.h    : 모터 구동부 선언
 │    ├── buttons.h     : 버튼 입력 및 상태 변화 선언
 │    ├── display.h     : OLED 화면 제어 선언
 │    └── modes.h       : 여러 동작 패턴(1~4번 모드) 선언
 └── src/               : 소스 폴더 (실제 펌웨어의 핵심 구현부)
      ├── actuator.cpp  : TB6612 드라이버 기반 PWM 제어 및 정지 명령 구현
      ├── buttons.cpp   : 버튼 꾹 눌림 방지(디바운스), 시작/정지/모드변경 상태 제어
      ├── display.cpp   : U8g2 라이브러리를 통한 화면 출력 및 인터페이스
      └── modes.cpp     : 모터들 4개의 타이밍 조합 및 블로킹 없는 smartDelay 루틴
```

## 2. 하드웨어 핀 할당 (Pin Map)

### [ TB6612 #1 - 액추에이터 1, 2 ]
* **AIN1_1** : 25
* **AIN2_1** : 26
* **BIN1_1** : 27
* **BIN2_1** : 14
* **PWMA_1** : 32
* **PWMB_1** : 33

### [ TB6612 #2 - 액추에이터 3, 4 ]
* **AIN1_2** : 13
* **AIN2_2** : 19
* **BIN1_2** : 18
* **BIN2_2** : 23
* **PWMA_2** : 4
* **PWMB_2** : 5
* **STBY**   : 15 (모터 드라이버 활성화 핀)

### [ 외부 컨트롤 및 I2C 통신 ]
* **BTN_START**  : 34 (장비 동작 시작)
* **BTN_STOP**   : 35 (장비 즉시 정지)
* **BTN_MODE**   : 39 (모드 1~4 순환 변경, 장비가 멈춰있을 때만 가능)
* **BTN_LED**    : 12 (MOSFET를 통한 조명 ON/OFF 전환)
* **LED_STATUS** : 2  (상태 확인용 내장/외장 LED)
* **LED_LIGHT**  : 0  (조명 제어용 MOSFET Gate 핀)
* **SDA_PIN**    : 21 (OLED I2C 데이터)
* **SCL_PIN**    : 22 (OLED I2C 클럭)

## 3. 핵심 작동 방식 원리 (알고리즘)

1. **상태 지향적 시스템 패턴**
   `handleButtons()` 함수가 끊임없이 호출되며 200ms 단위의 안전한 디바운싱 처리를 통해 버튼 변화를 감지합니다. 이 버튼들은 장비의 전체 상태(`isRunning`, `currentMode`, `ledLight`) 값만을 바꾸며, `loop()` 에서는 이 상태값을 보고 해당하는 모드를 실행합니다.

2. **비동기식 즉각 대응 제어 (`smartDelay`의 핵심 가치)**
   시퀀스를 만들 때 쓰이는 일반적인 `delay(800)` 함수 등은 동작 중 CPU를 잠재워버립니다. 
   이런 고착(블로킹) 문제를 회피하기 위해 `smartDelay()` 라는 획기적인 로직을 적용했습니다. 10ms 단위로 시간을 쪼개어 기다리는 동안에도 쉴새없이 사용자가 '정지(STOP)' 버튼을 눌렀는지(`!isRunning`) 확인합니다. 그래서 모터를 1초 동안 밀고 있는 상황이라 할지라도, 사용자가 정지 버튼을 누르면 남은 딜레이를 즉시 무시하고 루프를 탈출해 모터를 모두 안전하게 중단시킵니다.

3. **운동 모드 시나리오 (`src/modes.cpp`)**
   - **Mode 1 (Circular)**: 우측으로 모터를 한 번씩 돌아가며 밀고, 반대쪽을 당기는 방식으로 장치가 원을 그리도록 만듭니다.
   - **Mode 2 (Wok)**: 앞뒤 모터를 쌍으로 제어하여(1,3번 전진 및 2,4번 후진), 프라이팬 웍질을 하는 것처럼 리듬감 있는 직선 반동을 형성합니다.
   - **Mode 3 (Alternate)**: 대각선 쌍(1,4번과 2,3번)이 교차로 줄다리기 하듯 구동하여 크로스 혼합 동작을 생성합니다.
   - **Mode 4 (AI Mode)**: 추후 데이터 연동/센서 피드백 기반으로 동작을 동적으로 제어할 수 있는 확장용 슬롯입니다.
